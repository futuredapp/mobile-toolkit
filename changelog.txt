  🛠 Release 1.4.5 - security cleanup + tunnel hygiene

    🔐 Removed a go-ios tunnel identity file that was accidentally committed
    during development. The tunnel now stores its identity and pairing records
    in ~/.local/state instead of the current directory, so it can't happen again
    (this also avoids the macOS 26 lockdown-path block). Run "brew upgrade
    mobile-toolkit" to clear it from your install.

  🛠 Release 1.4.4 - now installable via Homebrew + irecord timing fix

    🍺 Install with "brew install" - no more editing your PATH by hand:
       brew tap futuredapp/mobile-toolkit
       brew trust futuredapp/mobile-toolkit
       brew install mobile-toolkit

    🎥 irecord now plays back in real time (it was compressing recordings of
    screens that change in bursts), and waits for the stream to be ready first

  🛠 Release 1.4.3 - groundwork for Homebrew installation

    📦 Runtime state (caches, device lists) now lives in ~/.local/state instead of
    inside the toolkit folder - so the toolkit can be installed read-only (Homebrew)

    🔜 Coming next: install via "brew install" - no more editing your PATH by hand

  🛠 Release 1.4.2 - over a dozen long-standing bugs fixed!

    🍏 go-ios is now installed as a pinned, checksum-verified release binary (v1.2.0)
    Existing installations are offered the update automatically - no Go toolchain needed anymore

    💿 Developer disk images are now mounted automatically via "go-ios image auto"
    The old "open Xcode and wait" workaround is gone

    📱 Device names of all modern iPhones and iPads are now resolved correctly
    (via Xcode's device database with an online fallback - no more raw board codes)

    🔪 ikill now relaunches the app on the correct device when several are connected

    📜 alog -f now uses logcat --pid for reliable filtering on current Android versions

    🔊 atalkback works again on current TalkBack versions and turns off cleanly

    🎥 irecord works again - on Apple Silicon and iOS 17+ (rebuilt on go-ios screenshot
    streaming, ~6 fps; use iquicktime for smooth high-fps footage; videosnap no longer needed)

    📹 arecord/isimulator recording fixes, aemulator setup fixes (JAVA_HOME, auth token)

    🔄 The update check no longer aborts the command you just ran

    ⭐️ Mobile Toolkit is now maintained by Futured (https://github.com/futuredapp)
    Built on the original work of Adam Svoboda - thanks, mate!
